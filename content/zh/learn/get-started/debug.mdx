# 调试

## 使用 ONNX  Runtime 的自定义构建

1. 按照[WebNN Installation Guide](installation)启用WebNNAPI并切换到Windows ML / ONNX Runtime 后端
2. 将 ONNX  Runtime 动态链接库放入 `C:\Program Files\` 下的文件夹中，例如`C:\Program Files\ONNXRuntime`
3. 使用开关 [`--webnn-ort-library-path-for-testing`](../../api-reference/browser-compatibility/chrome-flags#--webnn-ort-library-path-for-testing) 启动 Chrome 或 Edge 浏览器

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --webnn-ort-library-path-for-testing="C:\Program Files\ONNXRuntime" --allow-third-party-modules
```
</div>

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Microsoft\Edge SxS\Application\msedge.exe" --webnn-ort-library-path-for-testing="C:\Program Files\ONNXRuntime" --allow-third-party-modules
```
</div>

## 使用 Execution Provider 的自定义构建

如果您在 AMD、Intel、Nvidia 或 Qualcomm 设备上运行 WebNN，并且需要使用 ONNX  Runtime 的特定 Execution Provider 来调试 WebNN：

1. 按照[WebNN Installation Guide](installation)启用WebNNAPI并切换到Windows ML / ONNX Runtime 后端
2. 将 ONNX Runtime and Execution Provider 动态链接库放入 `C:\Program Files\` 下的文件夹中，例如`C:\Program Files\ONNXRuntime-EP`
3. 使用开关 [`--webnn-ort-ep-library-path-for-testing`](../../api-reference/browser-compatibility/chrome-flags#--webnn-ort-ep-library-path-for-testing) 启动 Chrome 或 Edge 浏览器

</div>

## 使用 GPU 或 NPU 驱动程序的自定义构建

如果您正在使用自定义或内部 GPU 或 NPU 驱动程序进行测试，请在启动浏览器时添加 [`--allow-third-party-modules`](../../api-reference/browser-compatibility/chrome-flags#--allow-third-party-modules) 开关。

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --allow-third-party-modules
```
</div>

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Microsoft\Edge SxS\Application\msedge.exe" --allow-third-party-modules
```
</div>

## 调试沙箱问题

在某些情况下，您可能需要通过使用 [`--disable-gpu-sandbox`](../../api-reference/browser-compatibility/chrome-flags#--disable-gpu-sandbox) 开关禁用 Chromium 的安全沙箱来调试 WebNN。

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --disable-gpu-sandbox
```
</div>

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Microsoft\Edge SxS\Application\msedge.exe" --disable-gpu-sandbox
```
</div>

## 检查详细日志记录

您可以使用开关 [`--webnn-ort-logging-level`](../../api-reference/browser-compatibility/chrome-flags#--webnn-ort-logging-level) 配置 ONNX  Runtime 的日志记录严重性级别。其他可用的严重级别包括“INFO”、“WARNING”、“ERROR”（默认）和“FATAL”。

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --webnn-ort-logging-level=VERBOSE
```
</div>

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Microsoft\Edge SxS\Application\msedge.exe" --webnn-ort-logging-level=VERBOSE
```
</div>
