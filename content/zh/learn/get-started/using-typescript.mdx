# 将 TypeScript 与 WebNN API 一起使用

TypeScript 为 JavaScript 项目提供强大的打字功能，通过代码完成和编译时错误检查等功能提高开发人员的工作效率。本指南介绍了如何在项目中集成 WebNN API 的 TypeScript 类型。

＃＃概述

[@webnn/types](https://www.npmjs.com/package/@webnn/types) 包包含 WebNN API 的 TypeScript 声明文件 ( `.d.ts` )，提供 WebNN 规范中使用的方法、属性和返回值的类型信息。

> **注意：**如果您使用 [Transformers.js](https://huggingface.co/docs/transformers.js/index) 或 [ONNX Runtime Web](https://onnxruntime.ai/docs/api/js/index.html) 等 JavaScript ML 库（其中包含它们自己的类型定义），则无需使用 `@webnn/types` 。

＃＃安装

使用您首选的包管理器将类型定义安装为开发依赖项：

```bash
# npm
npm install --save-dev @webnn/types
# yarn
yarn add --dev @webnn/types
# pnpm
pnpm add -D @webnn/types
```

＃＃配置

由于 `@webnn/types` 位于 DefinelyTyped [(@types)](https://github.com/DefinitelyTyped/DefinitelyTyped) 之外，您需要显式配置 TypeScript 来识别这些类型。您有两个选择：

### 选项1：在`tsconfig.json`中使用`types`

```json filename="tsconfig.json"
{
  "compilerOptions": {
    "types": ["@webnn/types"]
  }
}
```

### 选项2：在`tsconfig.json`中使用`typeRoots`

```json filename="tsconfig.json"
{
  "compilerOptions": {
    "typeRoots": [
      "./node_modules/@webnn/types",
      "./node_modules/@types"
    ]
  }
}
```

## 全局类型

配置完成后， WebNN 类型在您的项目中全局可用。您不需要显式导入它们 - TypeScript 会自动识别 WebNN 接口，例如 `navigator.ml` 和 `MLGraphBuilder` 。

```typescript
// No need to import WebNN types - they're globally available
async function runInference() {
  // TypeScript will recognize these types
  const ml:ML = navigator.ml;
  const context:MLContext = navigator.ml.createContext();
  const builder:MLGraphBuilder = new MLGraphBuilder(context);

  const input:MLOperand = builder.input('input', {dataType: 'float32', shape: [1, 3, 224, 224]});
  const weights:MLOperand = builder.constant({dataType: 'float32', shape: [16, 3, 3, 3]}, new Float32Array(432));

  // Types for convolution parameters
  const options:MLConv2dOptions = {
    padding: [1, 1, 1, 1],
    strides: [1, 1],
    dilations: [1, 1],
    groups: 1
  };
  
  const conv:MLOperand = builder.conv2d(input, weights, options);
  const graph:MLGraph = await builder.build({output: conv});
  
  // Rest of the inference code...
}
```

## WebNN TS 开发技巧

- 使用 TypeScript 的自动完成功能来发现可用的方法和属性
- 利用参数类型检查来避免常见的 WebNN API 使用错误
- TypeScript 接口将帮助您符合 WebNN 选项对象的预期形状
- 输入正确时，VSCode 和其他现代编辑器将显示 WebNN API 方法的文档工具提示
