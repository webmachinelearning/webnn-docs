## 自由覆盖面积

ONNX模型支持通过自由维度动态输入形状。自由维度允许模型接受沿该维度的不同大小的输入。例如，输入形状为`[batch, 3, height, width]`的模型处理任何数量、任何尺寸的RGB图像（3个可以）。

虽然很灵活，但当您的应用程序使用一致的输入大小时，您可以通过固定这些尺寸来优化性能。例如，如果您的 WebNN 应用程序始终处理单个 `224×224` 图像，您使用 `freeDimensionOverrides` 可以选项覆盖从 `[batch, 3, height, width]` 到 `[1, 3, 224, 224]` 的自由尺寸：

```js showLineNumbers filename="free-dimension-overrides.js"
const sessionOptions = {
  freeDimensionOverrides: {
    batch: 1,
    height: 224,
    width: 224
  }
};
```

此配置将输入维度锁定为特定值，从而可能提高模型性能。
