# ONNX2WebNN

[ONNX2WebNN](https://github.com/huningxin/onnx2webnn) 是一个命令行工具，可将 ONNX 模型转换为 WebNN JavaScript 文件，从而使用 WebNN API 在网络浏览器中启用机器学习推理。

## 概述

该工具生成两个主要输出：
- 包含 WebNN 模型定义的 JavaScript 文件 ( `.js` )
- 包含模型权重的二进制文件 ( `.bin` )

## 隐私和安全

**🔒 仅客户端处理**

所有模型转换和 WebNN JavaScript 文件生成操作仅在您的本地环境中运行。模型数据、权重或知识产权不会传输到或存储在任何外部服务器上。这使得该工具可以安全地用于：

- 专有型号
- 敏感知识产权
- 具有严格数据治理的企业环境
- 任何需要完整数据隐私的场景

## 安装

使用 pip 安装所需的依赖项：

```bash
pip install -U onnx protobuf numpy
```

## 命令行界面

### 句法

```bash
python onnx2webnn.py -if INPUT_ONNX_FILE_PATH -oj OUTPUT_JS_PATH [OPTIONS]
```

### 必需的参数

| Argument | Description |
|----------|-------------|
| `-if`, `--input_onnx_file_path` | Path to the input ONNX model file (`.onnx`) |
| `-oj`, `--output_js_path` | Path for the output WebNN JavaScript file (`.js`) |

### 可选参数

| Argument | Description |
|----------|-------------|
| `-nhwc`, `--nhwc` | Generate WebNN operators with NHWC input layout for `conv2d`, `convTranspose2d`, `resample2d`, and `pool2d` operations |
| `-json`, `--dump_json` | Export a JSON representation of the ONNX model for debugging |
| `-i`, `--json_indent` | Set indentation level for JSON output (default: 2) |
| `-h`, `--help` | Display help information |

## 基本用法

### 第 1 步：准备 ONNX 模型

> <InfoIcon /> 转换之前，使用 [onnxruntime_perf_test](https://github.com/microsoft/onnxruntime/blob/main/onnxruntime/test/perftest/README.md) 工具覆盖 ONNX 模型中的任何自由尺寸：

```bash
onnxruntime_perf_test -I -r 1 -u mobilenetv2-12-static.onnx -f batch_size:1 -o 1 mobilenetv2-12.onnx
```

### 第2步：创建输出目录

```bash
mkdir mobilenet
```

### 第 3 步：转换模型

```bash
python onnx2webnn.py -if ../sample_models/mobilenetv2-12-static.onnx -oj mobilenet/mobilenet.js
```

该命令在`mobilenet`目录中生成以下文件：
- `mobilenet.js` - WebNN 模型实现
- `mobilenet.bin` - 模型权重
- `mobilenet.json` - 模型元数据（可选）
- `index.html` - WebNN 型号的测试页

### 第 4 步：测试模型

通过 Node.js 启动本地 [HTTP server](https://www.npmjs.com/package/http-server) 并导航到测试页面：

```bash
http-server
```

在网络浏览器中打开 `http://localhost:8080/`。

## NHWC 布局支持

### 概述

ONNX 模型默认使用 NCHW（批次、通道、高度、宽度）布局。然而，一些 WebNN 后端（例如用于 CPU 和 GPU 的 TensorFlow Lite）在 NHWC（批量、高度、宽度、通道）布局下性能更好。

### 生成 NHWC 模型

添加 `-nhwc` 标志以生成针对 NHWC 布局优化的模型：

```bash
python onnx2webnn.py -if ../sample_models/mobilenetv2-12-static.onnx -oj mobilenet_nhwc/mobilenet_nhwc.js -nhwc
```

### 动态布局选择

使用 WebNN API 检测后端的首选布局并加载适当的模型：

```javascript
const deviceType = 'gpu'; // or 'cpu', 'npu'
const context = await navigator.ml.createContext({deviceType});
const layout = context.opSupportLimits().preferredInputLayout;

let webnnModel;
if (layout === 'nhwc') {
    webnnModel = new MobilenetNhwc();
} else {
    webnnModel = new Mobilenet();
}

// Initialize the model
await webnnModel.build({deviceType});
```

## 生成 QDQ WebNN 模型

该工具将 [QDQ (Quantize-Dequantize)](https://onnxruntime.ai/docs/performance/model-optimizations/quantization.html#onnx-quantization-representation-format) ONNX 模型转换为 WebNN 模型。

### 先决条件

QDQ 模型转换需要适当的张量形状信息来根据 [WebNN quantizeLinear and dequantizeLinear specification](https://www.w3.org/TR/webnn/#api-mlgraphbuilder-dequantizelinear) 处理尺度和零点张量重塑。WebNN 规范可能需要根据输入张量的秩和轴参数重塑这些张量。

#### 准备您的 ONNX 模型

在转换之前，请使用 [onnx-simplifier](https://github.com/daquexian/onnx-simplifier) 确保您的 ONNX 模型包含每个输出张量的形状信息：

1. 安装onnx -简化：
````外壳
pip3 安装 onnxsim
   ```

2. 简化您的模型：
````外壳
onnxsim 输入模型。onnx 输出_模型_简化。onnx
   ```

**例子：**
````外壳
onnxsim ../sample_models/mobilenetv2-12-qdq-static。onnx ../sample_models/mobilenetv2-12-qdq-static-简化。onnx
   ```

### 转换为 WebNN

#### 基本转换（NCHW 布局）

```shell
python onnx2webnn.py -if input_model.onnx -oj output_directory/output_model.js
```

**例子：**
```shell
python onnx2webnn.py -if ../sample_models/mobilenetv2-12-qdq-static-simplified.onnx -oj mobilenet_qdq/mobilenet_qdq.js
```

#### NHWC 布局转换

对于需要 NHWC 数据布局的模型，添加 `-nhwc` 标志：

```shell
python onnx2webnn.py -if input_model.onnx -oj output_directory/output_model.js -nhwc
```

**例子：**
```shell
python onnx2webnn.py -if ../sample_models/mobilenetv2-12-qdq-static-simplified.onnx -oj mobilenet_qdq_nhwc/mobilenet_qdq_nhwc.js -nhwc
```

### 参数

| Parameter | Description |
|-----------|-------------|
| `-if` | Input ONNX model file path |
| `-oj` | Output JavaScript file path for the WebNN model |
| `-nhwc` | (Optional) Use NHWC data layout instead of default NCHW |

## 使用 JSON 导出进行调试

将模型结构导出为 JSON 以便调试和分析：

```bash
python onnx2webnn.py -if ../sample_models/mobilenetv2-12-static.onnx -oj mobilenet/mobilenet.js -json
```

这将创建一个包含完整模型结构的附加 JSON 文件，可用于：
- 调试转换问题
- 了解模型架构
- 验证运算符映射

## 最佳实践

1. **模型准备**：在转换之前始终解析 ONNX 模型中的 [free dimensions](../onnx-runtime/free-dimension-overrides)
2. **布局选择**：使用动态布局检测来优化不同后端的性能
3. **测试**：在集成之前使用生成的 `index.html` 文件验证模型功能
4. **性能**：考虑生成 NCHW 和 NHWC 版本以获得最佳跨平台性能
