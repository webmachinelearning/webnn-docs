# WebNNUtils/OnnxConverter

[WebNNUtils/OnnxConverter](https://github.com/MicrosoftEdge/WebNNUtils/) 是一个 WebNN 编译器，用于从 .onnx 模型，使用 WebNN API 在网络浏览器中进行机器学习推理，无需框架开销。

## 概述

虽然 JS ML 框架可以评估浏览器中的模型，但它们涉及交付框架和昂贵的加载时间预处理，从而影响延迟。在预处理过程中，框架确定算子的输入形状，在 CPU 和加速执行之间划分算子，并优化模型图。

使用 OnnxConverter 就没有这样的开销。在编译时，OnnxConverter 采用静态 .onnx 文件并发出 JS 代码，该代码将构建等效的 WebNN 图表。生成的 JS 代码可以直接在浏览器中使用，无需任何额外的框架依赖。

## 隐私和安全

**🔒 仅客户端处理**

所有模型转换和 JavaScript 生成操作都专门在您的本地环境中运行。模型数据、权重或知识产权不会传输到或存储在任何外部服务器上。这使得该工具可以安全地用于：

- 专有型号
- 敏感知识产权
- 具有严格数据治理的企业环境
- 任何需要完整数据隐私的场景

## 生成的输出

OnnxConverter 生成几个文件：
- 包含 WebNN 图形构建函数的 JavaScript 文件 ( `.js` )
- 包含模型权重的一个或多个二进制文件 ( `.bin` )
- 用于调试的可选 JSON 元数据文件

核心输出是具有以下签名的 JavaScript 函数：

```javascript
function loadModelGraph(operand_input, weights_buffers, builder) {
    // Generated WebNN graph building code
    return output_operand; // or {output1: operand1, output2: operand2}
}
```

在哪里：
- `operand_input` - 模型的输入操作数
- `weights_buffers` - 包含模型权重的 ArrayBuffer 数组
- `builder` - MLGraphBuilder WebNN 对象

安装所需的 Python 依赖项：

```bash
pip install -U onnx protobuf numpy
```

## 命令行界面

### 句法

```bash
python OnnxConverter.py MODEL_FILE [OPTIONS]
```

### 必需的参数

| Argument | Description |
|----------|-------------|
| `model_file` | Path to the input ONNX model file (`.onnx`) |

### 可选参数

| Argument | Description |
|----------|-------------|
| `--weights_file` | Path to the output weights file (default: `weights.bin`) |
| `--output_file` | Path to the output JavaScript file (default: `model.js`) |
| `--max_weights_size` | Maximum size of a single weights file in MB (default: 1024) |

## 基本用法

### 第 1 步：转换模型

```bash
python OnnxConverter.py mobilenetv2-12.onnx --output_file mobilenet/mobilenet.js --weights_file mobilenet/weights.bin
```

该命令生成：
- `mobilenet/mobilenet.js` - WebNN 图形构建功能
- `mobilenet/weights0.bin` - 模型权重（可以分为多个文件）

### 第2步：后处理（必填）

生成的 JavaScript 需要后处理来处理代码排序和 CPU/GPU 分区：

```bash
# Reorder operations to ensure proper dependency resolution
python ReorderModel.py mobilenet/mobilenet.js

# Partition operations between CPU and GPU execution
python CPUGraphPartitioner.py mobilenet/mobilenet.js
```

### 第 3 步：在浏览器中加载权重

加载权重文件并将其传递给您的 WebNN 模型：

```javascript
const weights_file = 'weights0.bin';
let cache = await caches.open("weights");
let weights_response = await cache.match(weights_file);
if (!weights_response) {
    await cache.add(weights_file);
    weights_response = await cache.match(weights_file);
}
const weights_buffer = await weights_response.arrayBuffer();

// Create WebNN context and builder
const context = await navigator.ml.createContext({'deviceType': 'gpu'});
const builder = new MLGraphBuilder(context);

// Install CPU operations polyfill
InstallCpuOps(builder);

// Load the model graph
const model_output = loadModelGraph(input_operand, [weights_buffer], builder);
```

## 架构和处理流程

### 形状推断处理

OnnxConverter 不执行传统的形状推断。相反，它生成在 Runtime 处理动态形状的 WebNN 图形节点。WebNN 图形生成器通过可以使用 JavaScript 数字（不仅仅是张量）的 polyfilled 方法进行了增强。

例如，具有形状运算的 ONNX 图：
```javascript
// Pseudocode
a = tensor.shape();
b = a * 2;
```

生成调用 `shape()` 和 `mul()` 操作的 WebNN 节点，其中 `mul` 被填充以处理 `shape()` 返回的 JavaScript 数字。

### CPU 操作 Polyfill

在调用 `loadModelGraph` 之前，安装 CPU 操作 polyfill：

```javascript
const context = await navigator.ml.createContext({'deviceType': 'gpu'});
const builder = new MLGraphBuilder(context);
InstallCpuOps(builder);
```

polyfill（在 `CpuOps.js` 中定义）通过处理 JavaScript 数字的运算符增强了 `MLGraphBuilder`，从而支持形状操作和常量折叠等操作。

### 加工流水线

完整的转换过程分为三个阶段：

1. **OnnxConverter.py** - 发出 JS 图形构建代码的主编译器
2. **ReorderModel.py** - 修复了在需要后计算输入操作数的代码生成
3. **CPUGraphPartitioner.py** - 使用 `cpu_` 前缀注释需要 CPU 执行的操作

## 多文件重量支持

对于大型模型，权重可以自动拆分到多个文件中：

```bash
python OnnxConverter.py large_model.onnx --max_weights_size 512
```

这会创建多个必须全部加载的权重文件（ `weights0.bin` 、 `weights1.bin` 等）：

```javascript
const weight_files = ['weights0.bin', 'weights1.bin'];
const weights_buffers = [];

for (const file of weight_files) {
    const response = await fetch(file);
    weights_buffers.push(await response.arrayBuffer());
}

const model_output = loadModelGraph(input_operand, weights_buffers, builder);
```

## 支持的ONNX操作

OnnxConverter 支持一组全面的 ONNX 操作，包括：

### 核心业务
- **卷积**：Conv（带组、填充、步幅、膨胀）
- **激活**：Relu、Sigmoid、Softmax、LeakyRelu
- **池化**：GlobalAveragePool
- **算术**：加、减、乘、除、幂、开方

### 张量运算
- **形状操纵**：重塑、转置、松开、挤压
- **数据移动**：连接、收集、切片、扩展
- **比较**：等于、更少、哪里

### 高级操作
- **量化**：DequantizeLinear、DynamicQuantizeLinear、MatMulInteger
- **矩阵运算**：MatMul
- **专业**：DepthToSpace、ConstantOfShape、ReduceMean

### 软件实现的操作
一些操作是在 JavaScript 中实现的，而不是本机 WebNN ：
- **切片**：恒定切片的有限实现
- **挤压**：动态轴挤压
- **范围**：序列生成

## 调试与分析

### 代码生成问题

如果生成的代码存在依赖问题，请使用处理管道：

```bash
python OnnxConverter.py model.onnx --output_file model.js
python ReorderModel.py model.js
python CPUGraphPartitioner.py model.js
```

### 了解生成的代码

生成的 JavaScript 包含：
- 模型权重的常数定义
- WebNN 操作调用构建计算图
- 模型的输入/输出处理

## 最佳实践

1. **始终进行后处理**：在生成的代码上运行 ReorderModel.py 和 CPUGraphPartitioner.py
2. **安装Polyfills**：在加载模型之前使用InstallCpuOps()
3. **处理多个权重**：考虑大型模型中的分割权重文件
4. **缓存权重**：使用浏览器缓存权重文件以提高加载性能
5. **本地测试**：从 HTTP 服务器提供文件以实现正确的加载行为

## 局限性

- 没有动态形状推断（形状在 Runtime 解析）
- 复杂模型需要后处理管道
- 对某些 ONNX 操作的支持有限
- 某些操作类型需要 CPU 回退
