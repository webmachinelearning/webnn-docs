# 硬件错误
 
## 神经网络处理单元错误

### 在 WebNN NPU 上 Runtime ，NPU 利用率在 Windows 任务管理器中不可见

#### 问题

在 NPU 上运行 WebNN 应用程序时，NPU（神经处理单元）利用率不会出现在 Windows 任务管理器中，例如[Whisper Base on NPU](https://microsoft.github.io/webnn-developer-preview/demos/whisper-base/?devicetype=npu) ，即使安装了最新的 NPU 驱动程序后也是如此。

#### 原因

Windows 更新可能会在 Lenovo、ASUS、Dell 和 HP 等制造商的设备上自动将 Intel.com 驱动程序替换为 OEM 分发的旧版本。此行为会影响神经处理器（NPU、Intel AI Boost）和显卡驱动程序。

#### 现状及改进

预计 2026 年初发布的新 NPU 驱动程序将成为解决此问题的解决方案。

#### 替代方案：使用带有默认驱动程序的 WebNN 并通过开关 [`--disable_webnn_for_npu=0`](../../api-reference/browser-compatibility/chrome-flags#--disable_webnn_for_npu) 启动浏览器

在许多情况下，WebNN NPU 功能可与 Windows 更新附带的默认 OEM 提供的驱动程序配合使用。

如果稳定性比拥有绝对最新的驱动程序版本更重要，请使用开关 [`--disable_webnn_for_npu=0`](../../api-reference/browser-compatibility/chrome-flags#--disable_webnn_for_npu) 启动浏览器，以在 Chrome 或 Edge 中启用 WebNN NPU 功能。

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --disable_webnn_for_npu=0
```
</div>

<div className="ort-ep">
```shell
"%LOCALAPPDATA%\Microsoft\Edge SxS\Application\msedge.exe" --disable_webnn_for_npu=0
```
</div>

#### 参见

- [Intel NPU driver support documentation](https://www.intel.com/content/www/us/en/support.html)
- [Windows driver management documentation](https://learn.microsoft.com/en-us/windows-hardware/drivers/)

## GPU 错误

### 显卡驱动程序回滚影响 WebNN GPU 性能

#### 问题

Windows 更新会自动将图形驱动程序恢复到以前的版本，从而阻止 WebNN 实现最佳 GPU 性能。

#### 解决方案

参见[NPU solution](hardware-errors#cause)
