## WebNN 的架构如何？

WebNN API 使网络浏览器能够利用底层操作系统提供的本机机器学习功能。通过实现与硬件无关的抽象层，API 允许 JavaScript 框架访问高级机器学习功能，而不依赖于特定于平台的实现。WebNN 中的神经网络表示为由数学运算组成的计算图，构成计算机视觉、自然语言处理和其他机器学习应用的基础。

## 我可以在PC桌面上使用WebNN API吗？

WebNN API 在常见的计算设备和操作系统上运行。性能测试显示了以下方面的有效操作：

- 当前一代笔记本电脑（例如，配备 Intel Core Ultra 移动处理器的设备）
- 现代智能手机（例如 Google Pixel 3 类设备）

API 独立于平台，并与主要 ML 加速框架集成：

- Windows：Windows ML（ONNX运行时）
- Chrome操作系统：LiteRT
- Linux：LiteRT
- 安卓：LiteRT
- Apple 平台：Core ML

API 自动利用可用的硬件加速功能，包括 CPU 并行处理、GPU 计算功能和专用 ML 加速器，同时保持硬件抽象。开发人员可以调整性能参数，而无需管理特定的硬件细节。

另请参阅：[WebNN Browser Compatibility](../api-reference/browser-compatibility/api)

## 神经网络操作的正确抽象级别是什么？

神经网络运算是机器学习框架中使用的数学函数。现代框架支持大约 100 种标准运算（卷积、矩阵乘法、归约、归一化），某些框架还提供其他变体。

在设计WebNN运算时，我们考虑将高级函数分解为基本的数学运算。虽然这会减少已定义操作的数量，但会使网络变得更加冗长，并有失去硬件级优化的风险。现代平台为卷积和循环网络等常见操作提供内置优化。

因此，WebNN 实现了标准函数和构成规范中函数的所有较小操作。这保留了对特定于平台的优化的访问，同时允许使用分解操作构建新功能。

## 考虑过哪些替代方案？

WebGPU 提供了图形硬件上的网络 API 抽象，可用于实现 GPU 加速的神经网络操作。LiteRT 和 ONNX Runtime Web 等主要 JavaScript ML 框架目前使用 WebGPU 作为后端。一种替代方案是继续使用这些基于图形的框架来处理网络上的 ML 工作负载，而不是采用 WebNN。

然而，这种方法有两个明显的局限性：

1. 硬件优化有限

WebGPU 的图形抽象层无法充分利用特定于硬件的优化和操作系统级别可用的特殊指令。虽然 GPU 制造商不断创新特定于 ML 的硬件功能，但这些性能改进可能无法通过通用图形管道接口实现。

2. 一致性挑战

多样化的硬件生态系统和多个驱动版本使得在通过图形框架实现神经网络操作时很难确保不同设备上一致且准确的结果。传统上，操作系统擅长硬件一致性测试和质量保证，这是机器学习框架的关键功能，尤其是在医疗保健和工业流程等关键应用中，结果的准确性至关重要。

## 如何确定我是否在 Windows 11 上的 Windows ML / OpenVINO 执行提供程序上运行？

您可以按照以下步骤验证您正在使用哪个后端：

1.访问https://webnnreport.org/
2. 导航至 `chrome://histograms/` 并搜索 `WebNN`

### WebNN .ORT.WinAppRuntimeInstallState

找到`WebNN.ORT.WinAppRuntimeInstallState`并查看状态码：

```
  0: Windows Version Too Old    5: Paused 
  1: Activation Failure         6: Paused Low Battery
  2: Completed                  7: Paused WiFi Recommended
  3: Error                      8: Paused WiFi Required
  4: Canceled                   9: Runtime Already Present
```

```
0  ...

10 ... 
```

### WebNN .ORT.OpenVINOExecutionProvider.Status

找到`WebNN.ORT.OpenVINOExecutionProvider.Status`并查看状态码：

```
  0: Unknown                    3: Ensure ready failed

  2: Not installed
```

如果 Windows ML 运行时和 OpenVINO 执行提供程序无法正确安装，您可以手动安装它们。

### Windows App SDK 和执行提供程序手动安装

> <InfoIcon /> 在 Window 11 版本 24H2 或更高版本上，当您安装并启动浏览器时，Windows App SDK 和执行提供程序将在后台自动安装。
安装时间可能会根据您的网络速度而有所不同。如果 Windows App SDK 和执行提供程序未成功安装，请按照以下步骤操作。

<Steps>
#### 安装 Windows 应用程序 SDK

下载并安装 [Windows App SDK](https://learn.microsoft.com/en-us/windows/apps/windows-app-sdk/downloads#stable-release) 的最新稳定版本。

- 已知的最低兼容 Windows App SDK 版本：1.8.1

#### 通过 EP Catalog 可执行文件安装执行提供程序

1.访问[WebNN Execution Provider Catalog](https://github.com/ibelem/webnn-execution-provider-catalog/releases/tag/1.8_8wekyb3d8bbwe)
2.下载[ExecutionProviderCatalog.exe](https://github.com/ibelem/webnn-execution-provider-catalog/releases/download/1.8_8wekyb3d8bbwe/ExecutionProviderCatalog.exe)
3. 运行可执行文件以安装执行提供程序

```shell
C:\Users\webnn\Downloads>ExecutionProviderCatalog.exe
```
