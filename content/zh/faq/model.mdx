# 模型

## WebNN 应用的推荐模型尺寸是多少？

WebNN 应用程序的最佳模型大小取决于用户的网络条件和设备功能。虽然 WebNN 没有严格的大小限制，但较大的模型会影响下载时间和初始加载性能。
考虑实施 Web 应用程序功能来优化性能：

- 使用 service workers 启用离线功能和缓存模型
- 利用 Cache API 或 [Origin Private File System (OPFS)](https://web.dev/articles/origin-private-file-system) 在本地存储模型，减少后续加载时间

### ONNX Runtime Web 模型大小限制

ONNX Runtime Web 模型的大小范围很大，从几千字节到几 GB，但 ONNX 模型本身以 protobuf 格式序列化，最大大小为 2GB，而 ONNX Runtime Web 可以运行大小高达 4GB 的模型。

## 为什么不定义一个涵盖拓扑和权重的模型格式？

WebNN API 不直接支持模型格式，因为格式处理有意委托给框架和应用程序。这种设计决策保持了灵活性，同时允许框架实现自己的模型加载方法。
