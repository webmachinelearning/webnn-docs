# WebNN  基于 Chromium 的浏览器中的标志

<div className="flags">

## 旗帜

通过 `about://flags/` 访问

### `#web-machine-learning-neural-network`

|名称 |启用 WebNN API |
| :--: | :-------- |
|描述 |启用 Web 机器学习神经网络 ( WebNN ) API 。规范位于 https://www.w3.org/TR/webnn/ – Mac、Windows、Linux、ChromeOS、Android。|
|到期里程碑 |150 | 150
|命令行使用 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --enable-features=WebMachineLearningNeuralNetwork` |

* `%LOCALAPPDATA%`  表示  `C:\Users\<username>\AppData\Local\`

### `#webnn-onnxruntime`

|名称 |Windows ML / ONNX WebNN 的运行时后端 |
| :--: | :-------- |
|描述 |允许使用 ONNX 运行时通过 WebNN API 进行 CPU、GPU 和 NPU 推理。禁用此标志可以回退到 DirectML 或 TFLite。– Windows |
|到期里程碑 |146 | 146
|命令行使用 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --enable-features=WebMachineLearningNeuralNetwork,WebNNOnnxRuntime` |



### `#experimental-web-machine-learning-neural-network`

|名称 |启用实验性 WebNN API 功能 |
| :--: | :-------- |
|描述 |启用 Web 机器学习神经网络 ( WebNN ) API 中的附加实验性功能。需要启用“WebNNAPI”标志。– Mac、Windows、Linux、ChromeOS、Android。|
|到期里程碑 |150 | 150
|命令行使用 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --enable-features=EnablesExperimentalWebNNAPIFeatures` |

### `#webnn-coreml`

|名称 |WebNN 的核心 ML 后端 |
| :--: | :-------- |
|描述 |通过 WebNN API | 启用使用 Core ML 进行 GPU 和 NPU 推理
|到期里程碑 |146 | 146

### `#webnn-coreml-explicit-gpu-or-npu`

|名称 |WebNN 的 Core ML 后端（显式 GPU 或 NPU）|
| :--: | :-------- |
|描述 |允许使用 Core ML 通过 WebNN API | 进行显式 GPU 和 NPU 推理
|到期里程碑 |146 | 146

### `#webnn-directml`

|名称 |WebNN | DirectML 后端
| :--: | :-------- |
|描述 |通过 WebNN API 启用 DirectML 进行 GPU 和 NPU 推理。禁用此标志可以回退到 TFLite。– Windows。|
|到期里程碑 |146 | 146

## 开关（命令行标志）

### `--use-redist-dml`

|名称 |使用可再发行的 DirectML |
| :--: | :-------- |
|描述 |尝试使用可再发行的 DirectML.dll。用于在将较新的 DirectML 版本集成到 Windows 操作系统之前对其进行测试 WebNN。请查看有关 [DirectML releases](https://learn.microsoft.com/en-us/windows/ai/directml/dml-version-history) 的更多信息。|
|注意|DirectML 在 Microsoft Build 2025 期间被正式弃用。WebNN 将利用 Windows ML 访问 OpenVINO 和其他 EP 以获得硬件加速。|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --use-redist-dml` |

### `--disable_dml_meta_commands_for_gpu`

|名称 |禁用 GPU 的 DirectML 元命令 |
| :--: | :-------- |
|描述 |如果禁用元命令，请使用 DirectML 后备着色器运行 |
|注意|DirectML 在 Microsoft Build 2025 期间被正式弃用。WebNN 将利用 Windows ML 访问 OpenVINO 和其他 EP 以获得硬件加速。
|用法（真实）|`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --disable_dml_meta_commands_for_gpu=1` |
|用法（假）|  `"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --disable_dml_meta_commands_for_gpu=0` |

### `--disable_webnn_for_gpu`

|名称 |禁用 GPU 的 WebNN |
| :--: | :-------- |
|描述 |如果为 GPU 设备启用了 `--disable_webnn_for_gpu` 解决方法，我们需要在设置 WebNN GPU 功能状态之前检查是否有可用的 NPU 设备。|
|用法（真实）|`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --disable_webnn_for_gpu=1` |

### `--disable_webnn_for_npu`

|名称 |禁用 NPU 的 WebNN |
| :--: | :-------- |
|描述 |如果有 NPU 设备，请检查 `--disable_webnn_for_npu` 解决方法 |
|用法（真实）|`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --disable_webnn_for_npu=1` |
|用法（假）|  `"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --disable_webnn_for_npu=0` |

### `--webnn-coreml-dump-model`

|名称 |WebNN 核心 ML 转储模型 |
| :--: | :-------- |
|描述 |复制生成的 Core ML 模型 |
|注意|该文件夹需要可从 GPU 沙箱访问或使用 `--no-sandbox` |
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --no-sandbox --webnn-coreml-dump-model=/tmp/CoreMLModels` |

### `--webnn-tflite-dump-model`

|名称 |WebNN TFLite 转储模型 |
| :--: | :-------- |
|描述 |保存生成的TFLite模型文件 |
|注意|该文件夹需要可从 GPU 进程沙箱访问，或者必须使用 `--no-sandbox` |
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --no-sandbox --webnn-tflite-dump-model=/tmp/tflite_models` |

### `--webnn-ort-logging-level`

|名称 |WebNN ONNX 运行时日志记录级别 |
| :--: | :-------- |
|描述 |配置 ONNX 运行时 | 的日志记录严重性级别
|注意|“VERBOSE”，其他严重级别可能是“INFO”、“WARNING”、“ERROR”（默认）和“FATAL”|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --webnn-ort-logging-level=VERBOSE` |

### `--webnn-ort-dump-model`

|名称 |WebNN Windows ML 转储模型 |
| :--: | :-------- |
|描述 |在图级转换后保存优化的 ONNX 模型 |
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --no-sandbox --webnn-ort-dump-model=/tmp/ort_models` |

### `--webnn-ort-library-path-for-testing`

|名称 |WebNN ONNX 用于测试的运行时库路径 |
| :--: | :-------- |
|描述 |强制从开关指定的位置加载 `onnxruntime.dll` 以测试开发 ORT 构建。|
|注意|此开关不适用于运输场景，默认情况下会被忽略|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --webnn-ort-library-path-for-testing="C:\Program Files\ONNXRuntime" --allow-third-party-modules` |

### `--webnn-ort-ep-library-path-for-testing`

|名称 |WebNN ONNX 用于测试的运行时执行提供程序库路径 |
| :--: | :-------- |
|描述 |通过此开关指定 ORT EP 名称和库路径对，以测试开发 EP 版本。EP名称指定的ORT EP的库被强制从指定路径加载。该值的格式应为 `<ep_name>?<ep_library_path>` 。|
|注意|此开关不应在运输场景中使用，默认情况下会被忽略|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --webnn-ort-ep-library-path-for-testing=OpenVINOExecutionProvider?"C:\Program Files\ONNXRuntime-EP\onnxruntime_providers_openvino_plugin.dll" --allow-third-party-modules` |

### `--webnn-ort-ep-device`

|名称 |强制 ONNX 运行时仅选择特定执行提供程序 (EP) 设备 |
| :--: | :-------- |
|描述 |在 ORT 会话创建阶段匹配 WebNN 的给定 EP 名称、硬件供应商 ID 和硬件设备 ID。该值的格式应为： `ep_name` 、 `hardware_vendor_id` 、 `hardware_device_id` 。|
|注意|当`webnn-ort-logging-level`设置为`VERBOSE`或`INFO`时，可以在日志中查询可用的EP设备。如果没有找到匹配的设备，ORT 会话创建失败。所有条目必须按顺序提供，并且 hardware_vendor_id 和 hardware_device_id 都是十六进制字符串。|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --webnn-ort-ep-device=OpenVINOExecutionProvider,0x8086,0x4680` |

### `--webnn-ort-ignore-ep-blocklist`

|名称 |启用所有执行提供程序，忽略已启用标志阻止列表 |
| :--: | :-------- |
|描述 |控制是否启用执行提供程序。可以通过 `--webnn-ort-ignore-ep-blocklist` 命令行开关一次性启用禁用的执行提供程序。|
|注意|列出的 EP 必须与 `tools/metrics/histograms/metadata/webnn/histograms.xml` 中直方图变体 `WebNNOrtExecutionProvider` 的名称匹配。|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --webnn-ort-ignore-ep-blocklist` |

### `--webnn-ort-graph-optimization-level`

|名称 |WebNN ONNX 运行时图优化级别 |
| :--: | :-------- |
|描述 |配置ONNX运行时的图形优化级别|
|注意|级别可以是“DISABLE_ALL”、“BASIC”、“EXTENDED”和“ALL” |
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --webnn-ort-graph-optimization-level=DISABLE_ALL` |

### `--webnn-ort-enable-profiling`

|名称 |WebNN ONNX 运行时启用分析 |
| :--: | :-------- |
|描述 |收集 ORT 配置文件数据以进行性能分析。配置文件数据文件在 Chrome 的文件夹中生成，具有固定的命名格式 `prefix_date_time.json` .|
|注意|前缀可以由用户提供或使用`WebNNOrtProfile`作为默认|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --no-sandbox --webnn-ort-enable-profiling="WebNNOrtOvCpuProfile"` |

## 其他开关

### `--disable-gpu-sandbox`

|名称 |禁用 GPU 沙箱 |
| :--: | :-------- |
|描述 |通常用于修复与 GPU 进程相关的崩溃。仅应在开发或故障排除必要时使用，因为它会使应用程序容易受到攻击。|
|注意|这是一个不安全的配置。执行测试后，您应该退出 Chrome 并重新正常启动。|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --disable-gpu-sandbox` |

### `--no-sandbox`

|名称 |没有沙盒|
| :--: | :-------- |
|描述 |禁用 Chromium 的安全沙箱，该沙箱隔离浏览器进程以防止恶意代码执行。虽然有时在开发/测试环境中使用，但在常规浏览中使用此标志是危险的，因为它消除了基本的安全保护。|
|注意|这是一个不安全的配置。执行测试后，您应该退出 Chrome 并重新正常启动。|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --no-sandbox` |


### `--allow-third-party-modules`
|名称 |允许第三方模块 |
| :--: | :-------- |
|描述 |允许第三方软件将 DLL 或模块注入 Chrome 进程 |
|注意|虽然对于防病毒或辅助工具等合法软件很有用，但它可能会因允许潜在的恶意代码注入而带来安全风险。|
|用途 |`"%LOCALAPPDATA%\Google\Chrome SxS\Application\chrome.exe" --allow-third-party-modules` |
