# 浏览器兼容性

## WebNN API

> <InfoIcon />  2025 年 5 月 20 日：DirectML 在 Microsoft Build 2025 期间被正式弃用。WebNN  将利用 Windows ML 访问 OpenVINO 和其他 EP 以获得硬件加速。

- 后端选择顺序在  `webnn_context_provider_impl.cc`  中定义：Windows 首先尝试  ONNX Runtime，然后是 DirectML，然后是 TFLite 回退；Apple 在 TFLite 之前构建了 Core ML 尝试；所有其他平台均直接转至 TFLite。
- `features.gni`  在 Linux、ChromeOS、Android、Windows 和 Apple 上启用 TFLite ( `webnn_use_tflite` )； `webnn_use_chrome_ml_api`  控制对 Chrome ML GPU 委托的访问。
- TFLite 委托是可选的：如果请求的委托 (GPU/NPU) 丢失或失败，执行会透明地回退到 XNNPACK CPU 路径 (  `graph_impl_tflite.cc`  )。
- ONNX Runtime 支持当前需要 Windows 11 24H2+、 `kWebNNOnnxRuntime`  标志，并使用  `environment.cc`  中的  `execution-provider`  选择逻辑将适当的硬件 (GPU/NPU) 与 CPU 后备绑定。
- Core ML 通过调整  `MLModelConfiguration.computeUnits`  来尊重所请求的设备；如果没有  `kWebNNCoreMLExplicitGPUOrNPU`  ，GPU/NPU 请求默认为  `MLComputeUnitsAll`  (  `graph_impl_coreml.mm`  )。

### 笔记

- WebNN   API  主要支持 ChromeOS、Linux、macOS、Windows 和 Android 上基于 Chromium 的浏览器。
- 基于 Chromium 的浏览器包括但不限于 Google Chrome、Microsoft Edge、Opera、Vivaldi、Brave、Samsung Internet 等。
