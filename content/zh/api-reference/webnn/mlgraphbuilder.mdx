

# MLGraphBuilder

`MLGraphBuilder` 接口可以通过一组操作构建计算图。它管理图形构建会话期间的中间状态。

## 网络 IDL

```webidl showLineNumbers filename="web-idl"
typedef record<USVString, MLOperand> MLNamedOperands;

[SecureContext, Exposed=(Window, DedicatedWorker)]
interface MLGraphBuilder {
  // Construct the graph builder from the context.
  constructor(MLContext context);

  // Create an operand for a graph input.
  MLOperand input(USVString name, MLOperandDescriptor descriptor);

  // Create an operand for a graph constant.
  MLOperand constant(MLOperandDescriptor descriptor,
                     AllowSharedBufferSource buffer);

  // Create a scalar operand from the specified number of the specified type.
  MLOperand constant(MLOperandDataType type, MLNumber value);

  // Compile the graph up to the specified output operands asynchronously.
  Promise<MLGraph> build(MLNamedOperands outputs);
};
```

## 语法

```javascript
const builder = new MLGraphBuilder(context);

// Creating input and constant operands
const input = builder.input(name, descriptor);
const constant1 = builder.constant(descriptor, buffer);
const constant2 = builder.constant(type, value);

// Building the graph
const graph = await builder.build(outputs);
```

## 构造函数

### MLGraphBuilder ()

创建一个新的 `MLGraphBuilder` 实例。

```js showLineNumbers filename="constructor.js"
constructor(context)
```

＃＃＃＃ 参数

- `context` ：与图形生成器关联的 `MLContext` 对象。

＃＃ 特性

该接口不包含任何属性。

＃＃ 方法

＃＃＃ 输入（）

创建一个名为 `MLOperand` 的可用作图形的输入。

```js showLineNumbers filename="input.js"
builder.input(name, descriptor)
```

＃＃＃＃ 参数

- `name` ：表示输入名称的字符串。
- `descriptor` ：定义输入特征的 `MLOperandDescriptor` 对象。

#### 返回值

返回一个 `MLOperand` 对象。

＃＃＃ 持续的（）

创建常量 `MLOperand` 。该方法有两个重载：

#### 语法 1：创建图形常量

```js showLineNumbers filename="constant-1.js"
builder.constant(descriptor, buffer)
```

＃＃＃＃＃ 参数

- `descriptor` ：描述输出张量的 `MLOperandDescriptor`。
- `buffer` ：包含初始化数据的`AllowSharedBufferSource`。

##### 返回值

返回表示常量张量的 `MLOperand`。

#### 语法 2：创建标量操作数

```js showLineNumbers filename="constant-2.js"
builder.constant(type, value)
```

＃＃＃＃＃ 参数

- `type` ：指定数据类型的`MLOperandDataType`。
- `value` ：代表常数值的`MLNumber`。

##### 返回值

返回表示标量常数的 `MLOperand`。

＃＃＃ 建造（）

异步编译图形直至指定的输出操作数。

```js showLineNumbers filename="build.js"
builder.build(outputs)
```

＃＃＃＃ 参数

- `outputs` ：标识图形的输出操作数的 `MLNamedOperands` 记录。

#### 返回值

返回解析为 `MLGraph` 对象的 `Promise`。

## 实施注意事项

当构建 `MLContext` 类型设置为“默认”的图表时，会发生以下情况：

1.根据指定的输出操作数编译图
2. 图形初始化发生在 `MLGraph` 返回之前
3. 进行权重预处理，其中：
- 常量输入经过预处理
- 数据在操作系统级别缓存
- 此优化提高了后续图执行性能

## 示例

### 创建和构建一个简单的图表

```js showLineNumbers filename="example-1.js"
async function simpleGraph() {
  const context = await navigator.ml.createContext({
    deviceType: 'gpu'
  });
  // Create a graph builder
  const builder = new MLGraphBuilder(context);
  // Create an input
  const a = builder.input("a", {
    dataType: "float32",
    shape: [3, 4]
  });
  // Create a input
  const b = builder.input("b", {
    dataType: "float32",
    shape: [4, 3]
  });
  const c = builder.matmul(a, b);
  // Build the graph
  const graph = await builder.build({c});
  return graph;
}
```

### 创建标量常量

```js showLineNumbers filename="example-2.js"
const builder = new MLGraphBuilder(context);
const scalarConstant = builder.constant("float32", 42.0);
```

## 安全要求

1. 必须在安全上下文中使用（HTTPS 或 localhost）
2. 访问仅限于 Window 和 DedicatedWorker 上下文
3.图形构建器只有在其关联的`context`不丢失的情况下才能构建
