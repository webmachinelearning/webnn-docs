# MLContext

`MLContext` 接口表示神经网络计算工作负载和执行过程的全局状态。每个 `MLContext` 对象都有关联的 `context type` 、 `MLDeviceType` 和 `MLPowerPreference` 。

## 网络 IDL

```webidl showLineNumbers filename="web-idl"
typedef record<USVString, MLTensor> MLNamedTensors;

dictionary MLContextLostInfo {
  DOMString message;
};

[SecureContext, Exposed=(Window, DedicatedWorker)]
interface MLContext {
  undefined dispatch(MLGraph graph, MLNamedTensors inputs, MLNamedTensors outputs);

  Promise<MLTensor> createTensor(MLTensorDescriptor descriptor);

  Promise<ArrayBuffer> readTensor(MLTensor tensor);
  Promise<undefined> readTensor(MLTensor tensor, AllowSharedBufferSource outputData);

  undefined writeTensor(MLTensor tensor, AllowSharedBufferSource inputData);

  MLOpSupportLimits opSupportLimits();

  undefined destroy();

  readonly attribute Promise<MLContextLostInfo> lost;
};
```
## 上下文类型

上下文类型决定了执行上下文如何管理资源并促进神经网络图的编译和执行：

- `"default"`：根据用户首选项选项创建的上下文
- `"webgpu"` ：从 WebGPU 设备创建的上下文

当 `contextType` 设置为 `default` 且 `MLContextOptions.deviceType` 设置为 `"gpu"` 时，用户代理将创建一个在上下文中运行的内部 GPU 设备，并可以代表调用应用程序提交 ML 工作负载。

＃＃ 特性

### 丢失（只读）

返回一个在上下文丢失时解析的 `Promise`。当上下文的 `lost` `Promise` 被解决时，上下文被认为丢失。

- 类型：`Promise<undefined>`
- 投掷：无

＃＃ 方法

＃＃＃ 派遣（）

将已编译的 `MLGraph` 的计算工作量安排在 `MLContext` 的时间线上。

#### 语法

```js showLineNumbers filename="dispatch.js"
context.dispatch(graph, inputs, outputs)
```

＃＃＃＃ 参数

- `graph` ：表示要执行的计算图的 `MLGraph` 对象
- `inputs` ：包含计算图输入的 `MLNamedTensors` 对象
- `outputs` ：包含计算图输出的 `MLNamedTensors` 对象

#### 返回值

`undefined`

#### 注释

`dispatch()` 方法本身不提供图形执行已完成的信号。调用者应该 `await` 读回输出张量的结果。

### 创建张量()

创建与此 `MLContext` 关联的 `MLTensor` 。

#### 语法

```js showLineNumbers filename="create-tensor.js"
context.createTensor(descriptor)
```

＃＃＃＃ 参数

- `descriptor` : `MLTensorDescriptor` 对象

#### 返回值

返回解析为 `MLTensor` 的 `Promise`

### 读取张量()

此方法有两种变体：

#### 语法 1：（张量）

```js showLineNumbers filename="read-tensor-1.js"
context.readTensor(tensor)
```

将 `MLTensor` 的 `data` 从 `MLContext` 时间线读回脚本。

＃＃＃＃＃ 参数

- `tensor` : 要读取的 `MLTensor` 对象

##### 返回值

返回一个 `Promise`，该 `Promise` 解析为包含读取结果的 `ArrayBuffer`

#### 语法 2：（张量，输出数据）

```js showLineNumbers filename="read-tensor-2.js"
context.readTensor(tensor, outputData)
```

自带缓冲区变体，将 `MLTensor` 的 `data` 读回到提供的缓冲区中。

＃＃＃＃＃ 参数

- `tensor` : 要读取的 `MLTensor` 对象
- `outputData` ：将结果读入的 `AllowSharedBufferSource` 缓冲区

##### 返回值

返回解析为 `undefined` 的 `Promise`

### writeTensor()

将数据写入 `MLContext` 时间线上的 [​​[P1]] 的 `data`。 `MLTensor`

#### 语法

```js showLineNumbers filename="write-tensor.js"
context.writeTensor(tensor, inputData)
```

＃＃＃＃ 参数

- `tensor` ：要写入的 `MLTensor` 对象
- `inputData` ：`AllowSharedBufferSource` 缓冲区，其字节将被写入张量

#### 返回值

`undefined`

#### 注释

`writeTensor()` 方法本身不提供写入已完成的信号。调用者应该 `await` 读回张量的结果。

### opSupportLimits()

返回有关操作员级别的不同实现的支持级别的信息。

#### 语法

```js showLineNumbers filename="op-support-limits.js"
context.opSupportLimits()
```

#### 返回值

返回包含支持信息的 `MLOpSupportLimits` 对象。

#### MLOpSupportLimits 字典

```webidl showLineNumbers filename="web-idl"
dictionary MLOpSupportLimits {
  MLInputOperandLayout preferredInputLayout;
  MLSupportLimits input;
  MLSupportLimits constant;
  MLSupportLimits output;
};
```

##### 会员

- `preferredInputLayout` ：布局相关运算符的首选输入布局，例如 `conv2d()`
- `input` ：支持 `MLGraph` 输入 `MLOperand` 的限制
- `constant` ：支持 `MLGraph` 的常数 `MLOperand` s 的限制
- `output` ：支持 `MLGraph` 的输出 `MLOperand` 限制

#### MLSupportLimits 字典

```webidl showLineNumbers filename="web-idl"
dictionary MLSupportLimits {
  sequence dataTypes;
};
```

##### 会员

- `dataTypes` : 支持的数据类型

#### MLBinarySupportLimits 字典

```webidl showLineNumbers filename="web-idl"
dictionary MLBinarySupportLimits {
  MLSupportLimits a;
  MLSupportLimits b;
  MLSupportLimits output;
};
```

##### 会员

- `a` : `MLSupportLimits` 为操作数
- `b` : `MLSupportLimits` 用于 b 操作数
- `output` : `MLSupportLimits` 用于输出操作数

#### MLSingleInputSupportLimits 字典

```webidl showLineNumbers filename="web-idl"
dictionary MLSingleInputSupportLimits {
  MLSupportLimits input;
  MLSupportLimits output;
};
```

##### 会员

- `input` : `MLSupportLimits` 用于输入操作数
- `output` : `MLSupportLimits` 用于输出操作数

＃＃＃ 破坏（）

释放与上下文关联的所有资源。调用此方法后，任何挂起的计算请求和 MLTensor 操作（创建、读取或写入）都将失败。

#### 语法

```js showLineNumbers filename="destory.js"
context.destroy()
```

#### 返回值

- 类型：`undefined`

## 示例

### 使用 MLtensor 执行 MLGraph

```js showLineNumbers filename="example-1.js"
  const mlGraphExecution = async () => {
    const descriptor = {
      dataType: 'float32',
      shape: [2, 2]
    };
    const context = await navigator.ml.createContext({
      deviceType: 'gpu'
    });
    const builder = new MLGraphBuilder(context);

    // 1. Create a computational graph 'C = 0.2 * A + B'.
    const constant = builder.constant(descriptor, new Float32Array(4).fill(0.2));
    const A = builder.input('A', descriptor);
    const B = builder.input('B', descriptor);
    const C = builder.add(builder.mul(A, constant), B);

    // 2. Compile the graph.
    const graph = await builder.build({
      'C': C
    });

    // 3. Create reusable input and output tensors.
    const [inputTensorA, inputTensorB, outputTensorC] =
      await Promise.all([
        context.createTensor({
          dataType: A.dataType,
          shape: A.shape,
          writable: true
        }),
        context.createTensor({
          dataType: B.dataType,
          shape: B.shape,
          writable: true
        }),
        context.createTensor({
          dataType: C.dataType,
          shape: C.shape,
          readable: true
        })
      ]);

    // 4. Initialize the inputs.
    context.writeTensor(inputTensorA, new Float32Array(4).fill(1.0));
    context.writeTensor(inputTensorB, new Float32Array(4).fill(0.8));

    // 5. Execute the graph.
    const inputs = {
      'A': inputTensorA,
      'B': inputTensorB
    };
    const outputs = {
      'C': outputTensorC
    };
    context.dispatch(graph, inputs, outputs);

    // 6. Read back the computed result.
    const result = await context.readTensor(outputTensorC);
    console.log('Output value:', new Float32Array(result));
  }

  mlGraphExecution();
```

### 检查操作员支持限制

## 安全要求

- 必须在安全环境中使用
- 访问仅限于 Window 和 DedicatedWorker 上下文
- 需要正确处理共享缓冲区和张量数据，以防止内存泄漏和未经授权的访问
