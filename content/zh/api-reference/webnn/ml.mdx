# 机器学习

`ML` 接口提供了在 Web 应用程序中创建和管理机器学习上下文的方法。

## 网络 IDL

```idl showLineNumbers filename="web-idl"
enum MLDeviceType {
  "cpu",
  "gpu",
  "npu"
};

enum MLPowerPreference {
  "default",
  "high-performance",
  "low-power"
};

dictionary MLContextOptions {
  MLDeviceType deviceType = "cpu";
  MLPowerPreference powerPreference = "default";
};

[SecureContext, Exposed=(Window, DedicatedWorker)]
interface ML {
  Promise<MLContext> createContext(optional MLContextOptions options = {});
  Promise<MLContext> createContext(GPUDevice gpuDevice);
};
```

＃＃ 方法

### 创建上下文()

#### 语法

```js showLineNumbers filename="create-context.js"
createContext()
createContext(options)
createContext(gpuDevice)
```

＃＃＃＃ 参数

- `options` ：提供应用程序首选项的 `MLContextOptions` 对象
- `gpuDevice` ：与上下文一起使用的特定 `GPUDevice` 对象

#### 返回值

一个`MLContext`

＃＃ 描述

`ML` 接口支持创建具有特定设备和电源首选项的机器学习环境。它提供了两种创建上下文的方法 - 一种接受配置选项，另一种接受特定 GPU 设备。

### 设备类型选项

定义 ML 操作支持的设备类型的枚举：
- `"cpu"`：基于CPU的处理
- `"gpu"` ：基于 GPU 的处理
- `"npu"` ：基于神经处理单元的处理

#### `"cpu"`
- 提供跨设备的最大兼容性
- 基于硬件功能的可变性能
- 如果未指定则默认选项

#### `"gpu"`
- 提供最高的性能潜力
- 某些操作可能会退回到其他设备
- 需要兼容的图形硬件

#### `"npu"`
- 针对电源效率进行优化
- 适合持续的工作负载
- 可能会退回到其他设备来执行不受支持的操作

### 电源偏好选项

指定功耗首选项的枚举：
- `"default"` ：系统决定的行为
- `"high-performance"` ：速度优先于电源效率
- `"low-power"`：优先考虑电源效率而不是性能

#### `"default"`
- 让用户代理确定最佳行为
- 性能和功耗的平衡方法

#### `"high-performance"`
- 最大化执行速度
- 可能会增加功耗
- 适用于计算密集型任务

#### `"low-power"`
- 最大限度地减少功耗
- 可能会降低性能
- 非常适合电池供电的设备

### ML 上下文选项
ML 上下文的配置选项字典：

```webidl showLineNumbers filename="web-idl"
dictionary MLContextOptions {
  MLDeviceType deviceType = "cpu";
  MLPowerPreference powerPreference = "default";
}
```

## 示例

### 使用默认选项创建上下文

```js showLineNumbers filename="example-1.js"
try {
  const context = await navigator.ml.createContext();
  // Context ready for use
} catch (error) {
  console.error('Failed to create ML context:', error);
}
```

### 创建具有特定选项的上下文

```js showLineNumbers filename="example-2.js"
const options = {
  deviceType: "gpu",
  powerPreference: "high-performance"
};

try {
  const context = await navigator.ml.createContext(options);
  // High-performance GPU context ready
} catch (error) {
  console.error('Failed to create GPU context:', error);
}
```

### 与特定 GPU 设备一起使用

```js showLineNumbers filename="example-3.js"
// Assuming we have a GPUDevice instance
const gpuDevice = await navigator.gpu.requestAdapter()
  .then(adapter => adapter.requestDevice());

try {
  const context = await navigator.ml.createContext(gpuDevice);
  // Context created with specific GPU device
} catch (error) {
  console.error('Failed to create context with GPU device:', error);
}
```

## 安全要求
- 需要安全的上下文
- 在 Window 和 DedicatedWorker 上下文中可用

## 浏览器兼容性

> <InfoIcon /> `ML` 界面正在积极开发中，浏览器支持可能会有所不同。

参见[Browser Compatibility: WebNN API](browser-compatibility#webnn-apis)

## 发展现状

ML 接口和 MLContextOptions 正在积极开发中。目前的考虑因素包括：

- 实施回退设备机制
- 支持按首选顺序排列多个设备
- 设备排除功能
- 增强的错误处理
- 量化算子支持
