# MLOperand

`MLOperand`接口代表机器学习操作构建过程中的中间图节点。它可以表示常量、操作输入或神经网络图中组合多个操作的结果。

## MLPerandDescriptor 字典

`MLOperandDescriptor` 字典描述机器学习上下文中操作数的形状（维度）和数据类型。它用于指定 `MLGraph` 的输入和常量的特征，每个 `MLOperand` 包含一个内部 `MLOperandDescriptor` 。

它结合了两个重要信息：
- 操作数的数据类型（例如，float32、int8）
- 操作数的维度形状（例如，[批次、高度、宽度、通道]）

### 网络 IDL

```webidl showLineNumbers filename="web-idl"
enum MLInputOperandLayout {
  "nchw",  // Batch, Channels, Height, Width
  "nhwc"   // Batch, Height, Width, Channels
};

enum MLOperandDataType {
  "float32",
  "float16",
  "int32",
  "uint32",
  "int64",
  "uint64",
  "int8",
  "uint8"
};

dictionary MLOperandDescriptor {
  required MLOperandDataType dataType;
  required sequence<[EnforceRange] unsigned long> shape;
};
```

＃＃＃＃ 特性

##### `dataType`（必填）
- 类型：`MLOperandDataType`
- 说明：指定操作数的数据类型
- 可能的值：
- `"float32"` : 32 位浮点数
- `"float16"` : 16 位浮点数
- `"int32"` : 32 位有符号整数
- `"uint32"` : 32 位无符号整数
- `"int64"` : 64 位有符号整数
- `"uint64"` : 64 位无符号整数
- `"int8"` : 8 位有符号整数
- `"uint8"` : 8 位无符号整数

##### `shape`（必填）
- 类型：`sequence<[EnforceRange] unsigned long>`
- 描述：指定操作数维度的数组
- 对于标量操作数，该数组应该为空
- 每个元素代表一个维度的大小

#### 输入操作数布局类型

`MLInputOperandLayout` 枚举定义了可能的布局配置：`nchw` 和 `nhwc`

#### 示例

```js showLineNumbers filename="example-1.js"
const operandDescriptor = {
  dataType: "float32",
  shape: [1, 224, 224, 3]
};
```

## MLOperand 接口

`MLOperand` 接口提供对神经网络图中操作数的基本属性的访问。每个 `MLOperand` 都与一个 `MLGraphBuilder` 相关联，并包含定义其特征的内部描述符。

## 网络 IDL

```webidl showLineNumbers filename="web-idl"
[SecureContext, Exposed=(Window, DedicatedWorker)]
interface MLOperand {
  readonly attribute MLOperandDataType dataType;
  readonly attribute FrozenArray<unsigned long> shape;
};

dictionary MLOperatorOptions {
  USVString label = "";
};

typedef (bigint or unrestricted double) MLNumber;
```

＃＃＃ 特性

#### `dataType`（只读）
- 类型：`MLOperandDataType`
- 描述：返回操作数的数据类型
- 值源自操作数的内部描述符

#### `shape`（只读）
- 类型：`FrozenArray<unsigned long>`
- 描述：返回定义操作数形状的维度数组
- 标量操作数的空数组

## ML编号

`MLNumber` 类型是用于指定 `MLOperand` 运算的数字选项的类型定义。
它通过 `bigint` 和 `unrestricted double` 联合类型容纳大整数值和浮点数，以处理处理各种数据类型的 WebNN API 操作中的数字精度挑战。

＃＃＃ 描述

`MLNumber` 提供了一种灵活的数字类型，可以处理所有支持的 `MLOperandDataType` 值：
- 64 位整数（ `"uint64"` 和 `"int64"` ）
- 32位浮点（`"float32"`）
- 其他数字类型（ `"uint32"` 、 `"int32"` 等）

该实现在处理期间自动转换 `MLNumber` 值以匹配操作数的数据类型。

### 用例

`MLNumber` 可用于以下 `MLGraphBuilder` 方法：

- `constant()`（标量过载）
- `clamp()` (min/max options)
- `pad()`（填充值）
- ...

### 类型转换规则

1. 对于整数数据类型：
- 值被转换为适当的整数类型
- `bigint` 值保持完全精度
- 浮点值被截断

2. 对于浮点数据类型：
- 值被转换为适当的浮点精度
- `bigint` 值转换为浮点数

## 示例

```js showLineNumbers filename="example-2.js"
const context = await navigator.ml.createContext({
    deviceType: 'gpu'
});
const builder = new MLGraphBuilder(context);
const descriptor = {
    dataType: 'float32',
    shape: [1, 3, 224, 224]
};
const operand = builder.constant(descriptor, new Float32Array(3*224*224).fill(0.2));

console.log(descriptor.dataType); 
console.log(descriptor.shape); 
console.log(operand); 
```

## 安全要求

1. 必须在安全上下文中使用 ( `SecureContext` )
2. 访问权限仅限于：
- 窗口上下文
- 敬业的工人
3. 验证要求：
- 所有操作数必须由相同的`MLGraphBuilder`创建
- 形状尺寸必须是有效的无符号长整型值
- 数据类型必须符合操作要求

### 安全考虑

- 在操作前验证构建器关联和数值范围，检查操作中的数据类型兼容性
- 确保对大张量进行适当的内存管理，考虑大张量形状的内存影响
- 验证所有形状尺寸以防止内存不足的情况
- 对无效操作或无效描述符实施适当的错误处理
- 确保数据类型与模型的预期输入格式匹配
- 处理类型之间转换时潜在的精度损失
- 考虑数字溢出场景
