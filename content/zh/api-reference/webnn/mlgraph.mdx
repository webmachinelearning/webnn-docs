# MLGraph

`MLGraph` 接口代表一个已编译的计算图。一旦构建完成，已编译的计算图是不可变的，之后无法修改。

## Web IDL

```webidl showLineNumbers filename="web-idl"
[SecureContext, Exposed=(Window, DedicatedWorker)]
interface MLGraph {
  undefined destroy();
};
```

## 构造函数

`MLGraph` 接口没有构造函数。实例是通过其他 API 创建的。

## 属性

该接口不包含任何属性。

## 方法

### destroy()

释放与该图关联的所有资源。一旦调用该方法，该计算图将被标记为已销毁，并且不能再用于后续操作。

#### 语法

```js showLineNumbers filename="example-1.js"
graph.destroy()
```

#### 返回值

- `undefined`

## 注意事项

- 在构建之后，按规定计算图是不可变的，来保证计算的完整性。
- 所有输入和输出操作数必须通过它们各自的描述符进行正确描述。
- 实现细节由用户代理（User Agent）管理，不可直接访问。
- 销毁之后，不能再使用该图排队任何后续的工作负载。
- 一旦先前使用该计算图提交的所有工作负载都已完成，其底层的实现就可以释放与该图相关的任何额外的资源分配。

## 最佳实践

- 当不再需要图时，请始终调用 `destroy()` 以防止内存泄漏。
- 在调用 `destroy()` 之前，请确保对该计算图的所有操作已完成。
- 调用 `destroy()` 之后不要尝试使用计算图。

## 安全要求

`MLGraph` 接口：

1. 需要一个安全上下文 (`SecureContext`)
2. 仅暴露于 `Window` 和 `DedicatedWorker` 上下文
3. 构建后不可修改，以防止篡改计算图
4. 资源清理是异步执行的，以确保适当的内存管理
